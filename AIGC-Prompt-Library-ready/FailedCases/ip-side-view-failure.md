# IP 侧视图失败复盘

## 使用场景

适用于 IP 三视图中侧视图斜侧、双腿、重影或角色漂移的问题。

## 原始需求

从正面 IP 生成三视图时，侧视图没有严格 90 度，且出现多余肢体或角色造型变化。

## Prompt

失败描述：模型生成的侧视图不是纯侧面，而是 45 度斜侧；腿部出现两条腿、重影或透视穿插；角色脸型、头身比和服装细节发生漂移。修正 Prompt 应强调：纯侧视图必须为 strict 90-degree orthographic side view，只显示一条腿和一只脚，身体轮廓清晰简洁。保持角色脸型、五官比例、头身比、核心轮廓、服装、配色和表情完全一致，不进行任何重新设计。

## 负面词

不要斜侧、不要 45 度角、不要双腿、不要重影、不要透视穿插、不要露出另一只脚、不要改变角色设计。

## 修改指令

- 把 strict 90-degree orthographic side view 放进 Prompt 前半段。
- 明确 one leg and one foot only。
- 重新列出角色核心不变项，避免侧视图变成新角色。

## 注意事项

IP 侧视图是高失败率任务，建议单独生成侧视图，再和正背视图拼版。

## 效果评分

7.5/10：适合三视图失败修正。
